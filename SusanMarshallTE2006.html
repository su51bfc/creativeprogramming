The rationale behind my canvas project was based on the classroom code we had looked at during lectures. The development process began by looking at the different shapes that had been created in our classroom examples, and how some of the shapes had been animated. I thought about what I could create by breaking up the code to develop my own individual animation.

The original idea was to use the two spinning circles and add a rectangular box above and animate it to look like a car. I had difficulty creating this; the rectangle created would not appear on the canvas at the same time as the animated circles.

I decided to start from scratch with the code and look more closely at how shapes were created before starting any animation (Rowell, 2016). The theme chosen is a winter wonderland.

The canvas is based on the moving circle code from lecture (Rowell, 2016). I altered the size of the circles and repositioned them to create the shape of a snowman. Manipulating the circles allowed me to understand how each element of the arc() method worked. To create an arc with HTML5 Canvas, we can use the arc() method. Arcs are defined by a center point, a radius, a starting angle, an ending angle, and the drawing direction. Arcs can be styled with the lineWidth, strokeStyle, and lineCap properties. An arc a section of the circumference of an imaginary circle. This imaginary circle can be defined by x, y, and radius. We can define the arc itself with two points along the imaginary circle's circumference defined by startAngle and endAngle. These two angles are defined in radians and form imaginary lines that originate from the center of the circle and intersect the ends of the arc that we wish to create (Html5canvastutorials.com, 2016). The final argument of the arc method is antiClockwise which defines the direction of the arc path between its two ending points. Unless otherwise specified, this argument is defaulted to false (boolean value), which causes the arc to be drawn clockwise. Using Math.PI * 2 allows the drawing of a complete circle. Using a value higher will overdraw the circle a lower value will underdraw the circle.

The canvas is coloured blue using the fillRect() method to create a winter sky and a second rectangle at the bottom is the snow using the same method. I added the second rectangle (white) to demonstrate that I could create a shape, position it correctly on the canvas and apply style to it.

The buttons and nose were added to make the shape look like a snowman. The create the two circles for the eyes and positioned them correctly,variable posX was adjusted by -10 and + 10.

The text was added in two places to demonstrate that my abilities to place the code correctly have improved over the course of the task. The cloud was drawn to add another dimension to the canvas (Stackoverflow.com, 2016). It allowed me to look at the effect of the bezier curve and how changing the dimensions of the shape could change the clouds angles from soft curves to sharp angles.The bezierCurveTo() method adds a point to the current path by using the specified control points that represent a cubic BÃ©zier curve(W3schools.com, 2016).

The snowflakes were added as white circles, again using the method arc() and math.PI * 2 to draw complete circles; drawing these allowed me to understand better the co-ordinates of the canvas. Changing the co-ordinates of each circle made me aware of where on the canvas they would appear.

The onload function loads the canvas on the page and uses the getElementById property to change the content of the element in the DOM. To animate the canvas I declared a function and specified the functions name and type. The function moves the elements on the canvas and controls the speed of the movement. I played with the speed of the animation; commenting out certain parts of the code allowed me to understand how each element affected the objects on the canvas. This gave me confidence to break the code and examine how I could use it to create my final canvas.

In hindsight, my initial idea was unsuccessful because the code was not in the correct order on the canvas to allow the rectangle to appear either before or after the animation started. I discovered this the reason when I 'broke' my snowman code but the text stayed on the page. When looking at how I had broken the code I understood the importance of code placement. Moving the 'text' code allowed the text to remain constant on the canvas. This made me aware that when constructing the code it must be in the correct order
